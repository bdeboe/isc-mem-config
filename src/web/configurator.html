Global & Routine Buffers

Define how much of the memory available to IRIS you want to use to cache data & code.

% MB

IRIS uses the allocated Global Buffer Pool to keep global data in memory, shared between processes. It is sized relative to your available physical memory and preferably large enough to keep all "active" data in memory.
If you have databases with alternative block sizes, you may need to refine this in the generated script.

MB

Routine Buffers are used to keep application code cached between invocations and is also shared between processes. Except for very complex applications, a fixed allocation usually suffices.

Filling in the details

Finally, the General Memory Heap is used for other shared memory requirements, such as the locks and IPQ memory for parallel query execution. The remaining memory is used by individual processes for local variables, objects, etc.

MB

Overall amount of memory to allocate for the general memory heap.

MB

Locks also consume memory and are by definition visible across processes. You can allocate additional memory for locks (out of the heap memory defined earlier) if you expect a lot of concurrent transactions.

MB

You can also configure a maximum amount of memory for any individual IRIS process. The overall system default 256MB maintains a good balance between leaving room for important processes to claim what they need and keeping a safe maximum.
